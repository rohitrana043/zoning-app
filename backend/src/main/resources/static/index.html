Real Estate Zoning Update API

Backend service for managing and updating real estate zoning information

View Frontend App GitHub Repository

API Documentation

This service provides REST API endpoints for retrieving and updating real estate zoning information for parcels. The API is designed to support efficient data loading for map-based applications.

Base URL

https://zoning-app-production.up.railway.app/api

Available Endpoints

GET /parcels/geojson/bounds

Retrieve parcels within geographic bounds in GeoJSON format

Parameters:
  • north - Northern latitude bound
  • south - Southern latitude bound
  • east - Eastern longitude bound
  • west - Western longitude bound
Example:
/api/parcels/geojson/bounds?north=32.97283888431184&south=32.966358122355544&east=-96.77509260177614&west=-96.7867569923401

GET /parcels/clusters

Get clusters of parcels within bounds for efficient map rendering

Parameters:
  • north - Northern latitude bound
  • south - Southern latitude bound
  • east - Eastern longitude bound
  • west - Western longitude bound
  • zoom - Current map zoom level
Example:
/api/parcels/clusters?north=32.973676002537765&south=32.9607141259844&east=-96.7695393562317&west=-96.79286813735963&zoom=16

POST /parcels/update-zoning

Update zoning information for selected parcels

Request Body (JSON):
{
  "parcelIds": [1, 2, 3],
  "zoningType": "Residential",
  "zoningSubType": "Single Family",
  "username": "user"
}

GET /parcels/statistics

Get statistics about zoning types

Getting Started

To start using this API, you can:

  1. Visit the frontend application to see the API in action
  2. Explore the GitHub repository for the source code
  3. Test API endpoints using your preferred API client (Postman, cURL, etc.)
